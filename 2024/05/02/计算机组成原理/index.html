第一章 计算机系统概述

计算机系统简介

计算机系统的概念和分类

计算机系统由硬件和软件两大部分组成。

  • 所谓硬件即指计算机的实体部分，它由看得见摸得着的各种电子元器件、各类光、点、机设备的实物组成，如主机、外设等。

  • 所谓软件他是看不见摸不着的，由人们实现编制成具有各类特殊功能的信息组成。通常把这些信息，注入各类程序寄寓于各类媒体中，如RAM、ROM、磁带、磁盘、光盘、甚至纸带等。他们通常被作为计算机的主存或辅存的内容。

    计算机的软件通常可以分为两大类：系统软件和应用软件。

  • 系统软件又称为系统程序，主要用来管理整个计算机系统，监视服务，使系统资源得到合理调度，确保高效运行。它包括：标准程序库、语言处理程序（如将汇编语言翻译成机器语言的汇编程序；将高级语言翻译成机器语言的编译程序）、操作系统（如批处理系统、分时系统、实时系统）、服务性程序（如诊断程序、调试程序、连接程序等）、数据库管理系统、网络软件等等。

  • 应用软件又称为应用程序，它是用户根据任务需要所编制的各种程序。如科学计算程序，数据处理程序，过程控制程序，事务管理程序等。

计算机的发展

硬件发展

  1. 电子管和继电器时代（1940年代至1950年代）：早期的计算机使用电子管和继电器进行运算。这些设备庞大、耗能且不稳定，例如ENIAC就是一个典型的例子。虽然它们能够进行基本的计算，但是由于电子管容易损坏且需要频繁更换，因此效率很低。
  2. 集成电路的出现（1950年代至1960年代）：集成电路的发明使得电子元件能够集成在一块芯片上，大大提高了计算机的性能和稳定性。这一时期诞生了一些里程碑式的计算机，如IBM System/360。随着集成电路技术的进步，计算机的规模逐渐缩小，成本也逐渐下降。
  3. 个人计算机时代（1970年代至1980年代）：随着微处理器的发明，个人计算机开始普及。1975年，Altair 8800发布，标志着个人计算机的诞生。1977年，Apple II推出，进一步推动了个人计算机的发展。1981年，IBM发布了第一台IBM PC，开创了个人计算机的标准化时代。
  4. 移动计算和嵌入式系统（1990年代至今）：随着技术的进步，计算能力逐渐集成到更小、更便携的设备中。1990年代，个人数码助理（PDA）和智能手机开始出现。2007年，苹果公司发布了第一款iPhone，开启了智能手机时代。同时，嵌入式系统的应用范围也在不断扩大，从家用电器到汽车，再到工业自动化。

摩尔定律

摩尔定律是一项关于集成电路发展速度的经验观察，由英特尔公司的创始人之一戈登·摩尔（Gordon Moore）在1965年提出。该定律的核心内容是：集成电路上可容纳的晶体管数量，大约每隔18个月会翻一番，或者说其性能每隔18个月翻倍。

具体来说，摩尔定律可以总结为以下几点要点：

  1. 时间间隔：摩尔定律所描述的时间间隔为18个月，也有些人将其描述为每两年或每三年，但18个月是最常见的观察周期。
  2. 晶体管数量：摩尔定律关注的是集成电路上可容纳的晶体管数量。这意味着在相同的芯片面积上，晶体管数量将随时间呈指数级增长。
  3. 性能翻倍：随着晶体管数量的增加，集成电路的性能也会呈指数级增长，通常以性能翻倍的方式来衡量。
  4. 价格不变：摩尔定律假设在相同价格范围内，集成电路的晶体管数量会翻倍。这意味着虽然成本在增加，但每个晶体管的成本不会随之增加，从而使得每单位性能的成本越来越低。
  5. 应用范围：摩尔定律不仅适用于集成电路的晶体管数量，也可以推广到其他方面，如存储容量、传感器像素等。

摩尔定律的提出和持续验证推动了计算机和信息技术领域的快速发展。它促进了硅谷的兴起，推动了计算机性能的提升，使得计算机应用更加广泛和普及。然而，随着技术的发展逐渐接近物理极限，摩尔定律在未来可能面临挑战，需要寻找新的技术和方法来持续推动计算机技术的进步。

软件发展

  • 操作系统的演变

    • 早期操作系统（1950年代至1960年代）：简单的批处理系统和分时操作系统，如IBM的OS/360。
    • 个人计算机时代（1970年代至1980年代）：微软的DOS（Disk Operating System）成为个人计算机的主流操作系统。1985年，微软推出了Windows 1.0，开启了图形用户界面的时代。
    • 互联网时代（1990年代至今）：Windows、MacOS和Linux等操作系统不断演进，加入了更多的功能和安全性。同时，移动操作系统如iOS和Android也逐渐崛起。

  • 应用软件的发展

    • 办公软件：如Microsoft Office和Google Docs，使得办公工作更加高效。
    • 娱乐软件：游戏、多媒体软件和流媒体服务，如Steam、Netflix等，丰富了人们的娱乐生活。
    • 通讯软件：即时通讯应用（如WhatsApp、WeChat）、社交媒体平台（如Facebook、Twitter），改变了人们之间的沟通方式。

  • 编程语言和开发工具

    • 高级编程语言的发展（如C、Python、Java）使得软件开发更加简单和高效。
    • 集成开发环境（IDE）的出现，如Visual Studio、Eclipse等，为开发者提供了更好的开发体验和工作流程。

  • 人工智能和机器学习

    • 语音助手：如Apple的Siri、Amazon的Alexa和Google的Assistant，使得人机交互更加自然。
    • 智能推荐系统：网上购物、音乐和视频平台等的推荐系统利用机器学习算法来个性化推荐商品和内容。
    • 自动驾驶技术：基于人工智能和机器学习的自动驾驶技术正在逐渐成熟，有望改变交通运输行业。

计算机系统的层次结构

计算机系统的基本组成

计算机硬件

冯·诺伊曼计算机

冯·诺伊曼计算机是一种基于冯·诺伊曼结构设计的计算机体系结构，是现代计算机的基础模型之一。它由数学家约翰·冯·诺伊曼在二战期间提出，并在1945年发表了《报告书第一部分》（First Draft of a Report on the EDVAC）详细描述了这一概念。

冯·诺伊曼计算机的主要特点包括：

  1. 存储程序： 冯·诺伊曼计算机采用存储程序的概念，即程序和数据都存储在同一种内存中，并且以二进制形式表示。这使得程序能够被存储、读取和修改，从而实现了程序的自动执行。
  2. 单一总线结构： 冯·诺伊曼计算机采用了单一总线结构，将CPU、存储器和输入/输出设备连接在一条总线上，通过总线进行数据和控制信号的传输。
  3. 按顺序执行指令： 冯·诺伊曼计算机按照存储在内存中的指令序列依次执行，每条指令都包括操作码和操作数，操作码指示执行的操作类型，操作数则指定操作的对象。
  4. 分离存储器和处理器： 冯·诺伊曼计算机将存储器（内存）和处理器（CPU）分开，通过总线进行数据传输和交换，从而实现了数据和指令的有效分离。
  5. 运算单元和控制单元： 冯·诺伊曼计算机包含了运算单元（Arithmetic Logic Unit, ALU）和控制单元（Control Unit, CU），ALU负责执行算术和逻辑运算，而CU则负责从存储器中取出指令、解码并执行。

冯·诺伊曼计算机的提出标志着计算机科学的重要里程碑，它的基本原理被广泛应用于现代计算机体系结构的设计中，包括个人电脑、服务器、超级计算机等各种类型的计算机。

计算机的功能部件
  1. 存储器：用于存储数据和程序。包括内存（RAM）和长期存储设备（如硬盘驱动器、固态硬盘），内存用于临时存储程序和数据，长期存储设备则用于长期保存数据和程序。
  2. 运算器：执行计算机程序中的算术和逻辑运算。它包括算术逻辑单元（ALU），负责执行加减乘除等算术运算，以及逻辑运算，如与、或、非等。
  3. 控制器：从内存中获取指令，解码指令并控制数据流动。它包括控制单元（CU），负责协调各个部件的操作，确保程序按照指令顺利执行。
  4. 输入设备：用于输入数据和指令，包括键盘、鼠标、触摸屏、扫描仪和摄像头。这些设备允许用户向计算机提供信息。
  5. 输出设备：将计算机处理后的数据和结果展示给用户，包括显示器、打印机和音频设备（扬声器、耳机）。这些设备使用户能够获取计算机处理后的信息。

一般将运算器和控制器继集成到同一芯片上，成为中央处理器（CPU）。CPU和主存储器共同构成主机，而除主机外的其它硬件装置（外存、I/O设备等）统称为外部设备，简称外设。

计算机软件
软件的分类

软件按其功能分类，可分为系统软件和应用软件。

三个级别的计算机语言

计算机语言通常可以分为三个级别：机器语言、汇编语言和高级语言。

  1. 机器语言：
    • 定义：机器语言是计算机能直接理解和执行的语言，由二进制代码（0和1）表示。
    • 特点：与计算机硬件紧密相关，每个指令对应一组二进制码，难以阅读和编写，但计算机可以直接执行。
    • 示例：例如，对于某个特定计算机体系结构，将某个指令编码为特定的二进制序列，如00110110。
  2. 汇编语言：
    • 定义：汇编语言是机器语言的助记符表示法，使用助记符（如ADD、SUB等）代替二进制码，更易于理解和编写。
    • 特点：每个汇编语言指令对应一条机器语言指令，需要通过汇编器将汇编代码转换为机器语言。
    • 示例：将ADD指令翻译成对应的机器语言二进制码，如00011010。
  3. 高级语言：
    • 定义：高级语言是更接近人类语言的编程语言，使用类似英语的语法和关键字，更易于理解和编写。
    • 特点：与特定硬件无关，需要通过编译器或解释器将高级语言代码转换为机器语言。
    • 示例：C、Java、Python等都是高级语言，程序员可以使用这些语言编写复杂的程序，然后通过编译或解释器转换为机器语言。

这三个级别的计算机语言在不同层次上表达程序逻辑，越高级的语言越接近人类思维，编写和理解起来也更为方便，但执行效率可能相对较低；而低级语言则更接近计算机硬件，执行效率高，但编写和理解难度较大。

计算机硬件的基本结构

冯·诺伊曼计算机的基本结构包括：

  1. 中央处理单元（CPU）： 负责执行指令、处理数据和控制计算机的其他部件。
  2. 存储器（Memory）： 包括内存和外部存储器，用于存储数据和程序指令。
  3. 输入/输出设备（I/O Devices）： 用于与外部世界进行交互，如键盘、鼠标、显示器、打印机等。
  4. 控制单元（Control Unit）： 负责控制指令的执行顺序和数据的传输。
  5. 算术逻辑单元（ALU）： 执行算术运算（如加法、减法）和逻辑运算（如与、或、非）。
  6. 总线（Bus）： 用于传输数据、指令和控制信号的集合，包括地址总线、数据总线和控制总线。

这种结构的特点是指令和数据存储在同一存储器中，通过地址寻址来区分；程序和数据以二进制形式存储；指令和数据以相同格式存储在存储器中。

计算机软件和硬件的关系

计算机软件和硬件之间存在密切的关系，它们互相依存，共同构成了完整的计算机系统。

  1. 相互依赖：软件需要硬件的支持才能运行，而硬件则需要软件来控制和操作。没有合适的软件，硬件无法发挥作用；同样，没有硬件支持，软件也无法执行。
  2. 软件控制硬件：软件通过操作系统和驱动程序来控制硬件的各种功能和操作。例如，操作系统通过驱动程序来管理硬件设备，让它们能够与软件进行交互。
  3. 硬件支持软件：硬件提供计算和存储等基本功能，为软件的执行提供必要的计算资源和存储空间。不同类型的软件可能需要不同配置的硬件来运行。
  4. 性能和效率：软件的设计和编写可以充分利用硬件的性能，提高计算机系统的效率和性能。同时，硬件的性能和功能水平也会影响到软件的运行效果和体验。

总的来说，软件和硬件是相辅相成的，它们共同构成了计算机系统，软件决定了计算机的功能和特性，而硬件提供了软件运行的基础支持。

计算机系统的工作原理

源程序得到可执行程序

计算机将源代码转换为可执行程序的过程称为编译（Compilation）或解释（Interpretation），取决于编程语言和编译器/解释器的使用情况。下面是通常的步骤：

  1. 编写源代码：程序员使用文本编辑器或集成开发环境（IDE）编写源代码，其中包含程序的逻辑和功能。
  2. 编译或解释：
    • 编译：对于静态类型语言（如C、C++、Java），源代码需要通过编译器进行编译，生成与特定平台兼容的机器码或字节码。编译器会检查语法错误和类型错误，并将源代码转换为可执行程序。
    • 解释：对于动态类型语言（如Python、JavaScript），源代码由解释器逐行解释执行。解释器会将源代码逐行翻译为计算机能理解的指令，并实时执行代码。
  3. 链接（可选）：对于使用多个源文件的程序，编译器还可能进行链接操作，将不同源文件中的代码组合成一个可执行程序。链接器会解析函数调用、全局变量等引用，并将它们连接到对应的定义。
  4. 生成可执行文件：编译器或解释器将转换后的代码生成可执行文件，其中包含了机器码、字节码或解释器所需的中间代码。
  5. 运行程序：生成的可执行文件可以在相应的操作系统上运行，用户可以通过命令行或双击图标来启动程序，操作系统将加载可执行文件到内存中，并执行其中的指令。

这些步骤使得程序员能够将自己编写的源代码转换为计算机可以执行的可执行程序，从而实现程序的功能。

存储程序的基本思想

  1. 存储程序的基本思想是将计算机程序的指令和数据存储在计算机内存中，以便计算机可以按照程序的指令顺序执行操作。这个思想是现代计算机体系结构的核心之一，也是冯·诺伊曼体系结构的基础。

    冯·诺伊曼体系结构的基本思想包括：

    1. 存储器存储程序和数据：计算机的程序和数据被存储在内存中，程序和数据以二进制形式存储，可以随时读取和写入。
    2. 按顺序执行指令：计算机按照程序存储器中指令的顺序逐条执行指令，每条指令完成一个特定的操作，如算术运算、逻辑运算、数据传输等。
    3. 指令和数据共享存储器：程序和数据存储在同一存储器中，指令和数据可以交替存取，这种设计简化了计算机体系结构，提高了程序的灵活性和通用性。
    4. 自动顺序控制：计算机按照程序存储器中指令的顺序自动执行，不需要人工干预，程序的控制流由指令中的跳转指令和条件分支指令控制。
    5. 存储器地址寻址：计算机使用地址来访问存储器中的指令和数据，指令中的地址字段指示了下一条要执行的指令在存储器中的位置，数据也通过地址进行读取和写入。

    这些基本思想构成了现代计算机体系结构的基础，使得计算机能够高效地执行各种任务，从简单的算术运算到复杂的图形处理和人工智能任务。

计算机系统的多级层次结构

计算机系统的多级层次结构可以从不同的角度进行描述，包括硬件和软件两个方面：

  1. 硬件层次结构：
    • 物理层：包括处理器、内存、输入/输出设备、总线等实际的硬件组件。
    • 逻辑层：定义了硬件如何工作和交互，包括指令集架构（ISA）和处理器微架构等。
    • 组成层：由逻辑门、寄存器、ALU（算术逻辑单元）等组成，实现了基本的逻辑和算术操作。
    • 微体系结构层：定义了处理器的内部结构和组织，包括流水线、缓存层次结构等。
    • 电子层：描述了硬件的电子元件和电路结构，如晶体管、电容器等。
  2. 软件层次结构：
    • 应用层：包括用户直接使用的应用程序，如浏览器、文字处理软件等。
    • 操作系统层：管理计算机硬件资源和提供服务给应用程序，如Windows、macOS、Linux等操作系统。
    • 中间件层：提供在不同软件应用之间进行通信和交互的平台，如数据库管理系统（DBMS）、Web服务器等。
    • 编程语言层：定义了程序员编写软件的语法和语义，包括高级编程语言（如Python、Java）和低级编程语言（如汇编语言）。
    • 体系结构层：提供了软件和硬件之间的接口和抽象，使得软件可以跨平台运行，如虚拟机、编译器等。

这些层次结构相互交织，共同构成了计算机系统的复杂性。硬件层次提供了计算机系统的基本功能和性能，而软件层次则定义了如何利用硬件资源来实现各种应用和服务

计算机系统的工作原理

计算机系统的工作原理可以简单概括为：操作系统管理硬件资源，应用程序通过操作系统与硬件交互，CPU执行指令并处理数据，从而实现各种功能和任务。整个过程是一个不断循环的过程，用户输入数据和指令，计算机处理这些数据和指令，并输出处理结果，然后等待下一轮输入。

  1. 主存储器

    ​ 主存储器（简称主存或内存）包括了存储体M、各种逻辑部件以及控制电路等。存储体由许多存储单元组成，每个存储单元又包含若干个存储元件；每个存储元件只能存放一位二进制代码（0或者1）。这样，每个存储单元可以存储遗传二进制代码，这就被称为一个“存储字”；存储字的二进制位数称为“存储字长”。

    ​ 主存中的每个存储单元有一个唯一的编号，叫做存储单元的“地址”（Address）。主存的工作方式就是按照存储单元的地址，来实现对存储字各位的存（写入）取（读出）。这种存储方式叫做“按地址访问存储器”。

    ​ 为了实现按地址访问的方式，主存中还必须配置两个寄存器：MAR和MDR

    • MAR（存储器地址寄存器）：用来存放想要访问的存储单元的地址，它的位数决定了能访问的存储单元的最大个数。

    • MDR（存储器数据寄存器）：用来存放从存储单元中取出，或者准备向存储体单元存入的数据，他的位数和存储字相等。

      ​ 当然，如果想要完整的实现一个存储数据的操作，还需要CPU给主存加上各种控制信号，比如读命令、写命令以及地址编码驱动信号等。随着硬件技术的发展，主存现在都是统一制作的大规模集成电路芯片，所以一般都将MAR和MDR集成在CPU芯片中。

  2. 运算器

    运算器包括了一个算数逻辑单元（ALU）和最少三个寄存器。

    • ACC：累加器
    • MQ：乘商寄存器
    • X：操作数寄存器

    这三个寄存器在完成不同的算数运算时，所存放的操作数也各不相同。

    不同机器的运算器结构也有所不同，有的机器用MDR取代X寄存器。

  3. 控制器

    控制器由控制单元（CU）和程序计数器（PC）、指令寄存器（IR）组成。

    PC：程序计数器，用来存放当前将要执行指令的地址。他与主存的MAR之间有一条直接通路，且具有自动加1的功能，也就时可以自动形成下一条指令的地址。

    IR：指令寄存器，用来存放当前的指令。IR的内容来自主存的MDR，包含了操作码和地址码。IR中的操作码OP（IR）会送至CU，可以记作OP（IR）->CU，用来分析指令；而地址码Ad(IR)作为操作数的地址送至储存器的MAR，可以记作Ad（IR）->MAR。

    CU是控制器的核心组件，用来分析当前指令所需完成的操作，并发出各种位操作命令序列，从而控制所有被控对象。控制器是计算机的神经中枢，由他指挥各部件自动协调地工作；完成一条指令操作，需要取指、分析和执行三个阶段。

  4. I/O

    I/O系统包括各种输入输出设备及其响应的接口。每一种I/O设备都由I/O接口与主机联系，他接收CU发出的各种控制命令，并完成相应的操作。

计算机性能指标

衡量一台计算机的性能室友多项技术指标综合确定的，既包含硬件的各类性能，又包括软件的各种功能。

主要性能指标

  1. 机器字长

    ​ 机器字长，是指CPU一次能处理数据的位数，也就是CPU内部用于整数运算的数据通路的宽度。字长通常就等于CPU的通用寄存器宽度，也就是CPU内用于整数运算的运算的运算器的位数，它反映了计算机处理信息的能力。我们平常所说“一台16位或32位或64位的机器”，这里的16，32，64就是指字长。

    ​ 字长越长，数的表示范围也越大，精度也越高。机器的字长也会影响机器的运算速度。倘若CPU字长较短，又要运算位数较多的数据，那么需要经过两次或多次的运算才能完成，这样势必影响征集的运行速度。当然，机器字长对于硬件的造价也有较大的影响。他将直接影响加法器（或ALU）、数据总线以及存储字长的位数。

    • 指令字长：一条指令中包含的二进制代码的位数。

    • 存储字长：一个存储单元中存储的二进制代码的长度。

      ​ 指令字长和存储字长，都必须是字节（byte）的整数倍。指令字长一般取存储字长的整数倍：如果指令字长等于存储字长的2倍，那么取一条指令就需要2个机器周期；如果指令字长等于存储字长，那么取指周期就等于机器周期。

  2. 数据通路带宽

    ​ 数据通路带宽，是指数据总线一次所能并行传送信息的位数，他关系到数据的传送能力。这里所说的数据通路带宽是指外部数据总线的宽度，它与CPU内部的数据总线宽度（机器字长）可能不同。

  3. 存储容量

    ​ 存储器的容量，包括主容量和辅存容量。我们一般主要关心主存容量。

    ​ 主存容量是指主存中所能存储信息（二进制代码）的最大容量，通常以字节数来衡量。

    ​ 存储容量 = 存储单元个数 X 存储字长

    ​ 在主存储器中，MAR的位数反应了存储单元的个数，MDR的位数则反应了存储字的长度。例如，MAR为16位，表示2^16^ = 65536，也就是说对应的存储体内有65536个存储单元（一般称为64K内存，1K=1024）；而如果MDR为32位，那么主存的存储容量为2^16^ X 32 = 2^21^ = 2M位 （1M = 2^20^)。

    ​ 现代计算机中常以字节的个数来描述容量的大小，一个字节 （Byte）被定义为8位二进制代码。所以上述存储容量是2M位 （bit），也可以用2^18^字节 （Byte，简写为B）表示，记作2^18^B或256KB。

    ​ 同理，辅存容量也可以用字节数来表示，例如，某机辅存（如硬盘）容量为128GB （1G = 1KM = 2^30^）。

  4. 运算速度

    ​ 计算机的运算速度与许多因素有关，如机器的主频、CPU的结构、执行什么样的操作、主存本身的速度（主存速度快，取指、取数就快）等等都有关。

专业术语解释

吞吐量：系统在单位时间内处理请求的数量，主要取决于主存的存取周期。

响应时间：从用户向计算机发送一个请求，到系统对该请求做出响应并获得所需结果的时间。通常包括CPU时间（计算机执行程序的时间）和等待时间（用于磁盘访问、存储器访问、I/O操作等的时间）。

主频（CPU的时钟频率）：机器内部主时钟的频率，他是衡量机器速度的重要参数。对统一型号的计算机，主频越高，完成指令的一个步骤所用的时间越短，执行指令的速度越快。通常以赫兹（Hz）为单位。

CPU时钟周期：节拍脉冲的宽度或周期，也就是主频的倒数，他是CPU中最小的时间单位。

​ CPU时钟周期 = 1/主频

CPI：执行一条指令所需的时钟周期数。对一个程序或一台机器来说，CPI指的是该程序或该机器指令集中所有指令执行所需要的平均时钟周期数。

CPU执行时间：运行一个程序所花费的时间。

​ CPU执行时间 = （指令数 x CPI ) / 主频

对于同一个程序，CPU的执行时间就代表了CPU的性能，它主要取决于三个要素：主频、CPI和指令数。这三者是相互制约的。不同的机器可以有不同的指令集，更改指令集可以让程序的指令数更少，但CPI可能就会增大；同时可能引起CPU结构的调整，从而造成主频的降低。
